O estado (state) de toda a sua aplicação é armazenado em uma árvore de objetos dentro de uma única reserva (store).

State is read-only

A única maneira de mudar o estado (state) é emitir uma ação (action) com um objeto que descreve o que acontecerá.

Redux Action


store.dispatch({
  type: 'Job/SAVE', // action type
  model: { name: 'Game of Thrones' },
});

// with action creator
function saveJob(model) {
  return {
    type: 'Job/SAVE', // action type
    model,
  };
}
store.dispatch(saveJob({ name: 'Game of Thrones' });
					

// with action creator
function saveJob(model) {
  try {
    // do something, like validation

    return { type: 'Job/SAVE', model };
  } catch (error) {
    return { type: 'VALIDATION_ERROR', error };
  }
}
store.dispatch(saveJob({ name: 'Game of Thrones' });

Changes are made with pure functions

Para especificar como a árvore de estados (store) é transformada por ações (actions), você escreve redutores puros (pure reducers).

Reducer:


function counter(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1
    case 'DECREMENT':
      return state - 1
    default:
      return state
  }
}

Porque imutabilidade é importante

  • Facilidade em ir e vir na Viagem no Tempo
  • Acompanhamento das mudanças
  • Determinar quando re-renderizar no React
